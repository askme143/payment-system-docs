Sequence Diagram

신규 회원 최초 구독 성공 플로우

체크아웃 생성, 빌링 인증, 빌링키 발급, 첫 자동결제 승인, 웹훅 보정까지의 성공 흐름입니다.

전체 시퀀스 목록전체 API 목록API 상세 설명이메일 발송 시스템 아키텍처 문서MongoDB 구조 문서결제 시스템 잠재 위험 및 대응 문서

사용 API

GET /plans

구매 가능한 상품과 구독 플랜 목록을 조회합니다.

GET /plans/{planId}

단일 플랜의 가격, 주기, 판매 가능 상태를 조회합니다.

POST /subscriptions/checkout

구독 빌링 인증 전에 구독 대기 레코드와 토스 SDK 호출 데이터를 생성합니다.

POST /subscriptions/confirm

프론트 성공 페이지 요청을 받은 백엔드 서버가 호출하며, 빌링키 발급과 첫 결제로 구독을 활성화합니다.

POST /webhooks/toss-payments

토스페이먼츠 결제 이벤트를 수신하고 내부 결제/인보이스/구독 상태를 보정합니다.

GET /subscriptions/me

현재 회원의 상품별 구독 목록, 이용 기간, 다음 결제일, 공통 기본 결제수단 표시 정보를 조회합니다.

연관 잠재 위험
  • provider-paid-internal-unpaid PG 승인 성공 후 내부 미반영
  • duplicate-active-subscription 상품별 활성 구독 중복 생성
  • webhook-out-of-order-or-duplicate 웹훅 중복 또는 순서 역전
