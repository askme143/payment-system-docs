Sequence Diagram

구독 결제 최종 실패 후 구독 종료 플로우

정기 결제 재시도가 모두 실패하면 구독을 canceled로 종료하고, 다시 이용하려면 새 구독을 시작하도록 안내하는 흐름입니다.

전체 시퀀스 목록전체 API 목록API 상세 설명MongoDB 구조 문서결제 시스템 잠재 위험 및 대응 문서

사용 API

GET /invoices/{invoiceId}

단일 인보이스의 금액, 결제 시도, 실패 사유, 영수증 정보를 조회합니다.

POST /internal/subscription-billing/{invoiceId}/retry

실패한 구독 결제를 수동 또는 배치로 재시도합니다.

최종 실패 후 즉시 구독 종료

마지막 자동 재시도까지 실패하면 인보이스를 final_failed로 확정하고 구독을 canceled로 종료한 뒤 재구독 안내 메일을 발송합니다.

시점구독 상태결제 상태설명
최종 실패 전past_duefinal_failed마지막 재시도가 실패했고 구독 종료 조건을 검증합니다.
구독 종료 후canceledfinal_failed구독은 즉시 종료되고 다음 과금일은 제거됩니다.
재이용new subscription required-다시 이용하려면 기존 구독 복구가 아니라 새 구독을 시작합니다.
