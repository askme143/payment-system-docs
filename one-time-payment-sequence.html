일반결제 플로우

구매 의도 생성, 결제 시도 생성, 토스 결제창 인증, 결제 승인, 웹훅 보정까지의 일반결제 성공 흐름입니다. 실패 후 재시도는 기존 시도를 재사용하지 않고 새 paymentId와 orderId를 발급합니다.

전체 시퀀스 목록전체 API 목록API 상세 설명

사용 API

POST /payments/orders

일반결제 결제창 호출 전에 내부 주문을 생성합니다.

POST /payments/confirm

토스 결제 인증 후 일반결제를 최종 승인합니다.

GET /payments/{paymentId}

내부 결제 상태와 토스 승인 결과를 조회합니다.

POST /webhooks/toss-payments

토스페이먼츠 결제 이벤트를 수신하고 내부 결제/구독 상태를 보정합니다.

일반결제 성공

회원의 구매 의도(checkoutId) 아래 첫 결제 시도 또는 재시도 시도를 생성하고, 토스 인증과 서버 승인을 거쳐 결제를 완료하는 흐름입니다.

상태 요약

시점구독 상태결제 상태설명
구매 의도 생성 후-checkout_opencheckoutId가 생성되고 첫 결제 시도를 만들 수 있습니다.
결제 시도 생성 후-readypaymentId와 orderId가 생성되어 토스 결제창 인증을 시작할 수 있습니다.
토스 인증 성공 후-authorizedpaymentKey, orderId, amount가 프론트 성공 페이지로 돌아왔지만 서버 승인은 아직 완료되지 않았습니다.
결제 승인 성공 후-paid현재 checkoutId의 최종 성공 결제 시도가 확정되고 이전 실패 시도들은 이력으로 남습니다.
