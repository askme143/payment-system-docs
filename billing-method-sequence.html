결제수단 관리 플로우

회원이 결제 수단을 추가하고, 모든 상품 구독이 함께 사용하는 공통 기본 결제수단을 지정하며, 활성 구독이 1개 이상 있으면 최소 1개의 결제수단을 유지하도록 삭제를 제한하는 흐름입니다.

전체 시퀀스 목록전체 API 목록API 상세 설명결제 시스템 아키텍처 문서MongoDB 구조 문서결제 시스템 잠재 위험 및 대응 문서

사용 API

POST /billing/auth

기존 회원이 결제수단을 변경할 때 빌링 인증 시작 데이터를 생성합니다.

POST /billing/issue

빌링 인증 성공 후 새 빌링키를 발급받아 결제수단으로 저장합니다.

GET /billing/methods

회원이 등록한 표시용 결제수단 목록을 조회합니다.

PATCH /billing/methods/{billingMethodId}/default

활성 결제수단 중 하나를 기본 결제수단으로 지정합니다.

DELETE /billing/methods/{billingMethodId}

사용하지 않는 결제수단을 삭제 또는 비활성화합니다.

연관 잠재 위험
  • billing-method-default-race 기본 결제수단 변경과 과금 경합

상태 요약

시점구독 상태결제 상태설명
추가 인증 시작 후activebilling_auth_pending회원은 기존 구독을 유지한 채 새 카드 빌링 인증을 진행합니다.
첫 결제수단 저장 후activebilling_method_active저장된 결제수단이 자동으로 기본 결제수단이 됩니다.
추가 결제수단 저장 후activebilling_method_active요청 정책에 따라 새 결제수단을 기본값으로 지정하거나 기존 기본값을 유지합니다.
기본값 변경 전activebilling_method_active기존 기본 결제수단이 모든 상품 구독의 다음 정기결제 대상으로 지정되어 있습니다.
기본값 변경 후activebilling_method_active새 기본 결제수단이 모든 상품 구독의 다음 정기결제 대상으로 지정되고 기존 결제수단은 활성 상태로 남습니다.
삭제 요청 전activebilling_method_active모든 활성 상품 구독의 다음 결제에 사용할 공통 기본 결제수단이 존재합니다.
마지막 결제수단 삭제 시도activebilling_method_required구독 유지를 위해 삭제를 거부하고 결제수단 추가를 안내합니다.
보조 결제수단 삭제 후activebilling_method_active삭제 대상만 비활성화되고 기본 결제수단은 그대로 유지됩니다.
