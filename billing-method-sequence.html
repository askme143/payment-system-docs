결제수단 관리 플로우

회원이 결제 수단을 추가하고, 모든 상품 구독이 함께 사용하는 공통 기본 결제수단을 지정하며, 활성 구독이 1개 이상 있으면 최소 1개의 결제수단을 유지하도록 삭제를 제한하는 흐름입니다.

전체 시퀀스 목록전체 API 목록API 상세 설명

사용 API

POST /billing/auth

기존 회원이 결제수단을 변경할 때 빌링 인증 시작 데이터를 생성합니다.

POST /billing/issue

빌링 인증 성공 후 새 빌링키를 발급받아 결제수단으로 저장합니다.

GET /billing/methods

회원이 등록한 표시용 결제수단 목록을 조회합니다.

PATCH /billing/methods/{billingMethodId}/default

활성 결제수단 중 하나를 기본 결제수단으로 지정합니다.

DELETE /billing/methods/{billingMethodId}

사용하지 않는 결제수단을 삭제 또는 비활성화합니다.

결제 수단 추가

회원이 새 카드를 빌링 인증하고, 서버가 토스에서 새 빌링키를 발급받아 결제수단으로 저장하는 흐름입니다.

기본 결제수단 지정

활성 결제수단 중 하나를 모든 상품 구독의 다음 정기결제에 사용할 공통 기본 결제수단으로 지정하는 흐름입니다.

결제 수단 삭제

회원이 사용하지 않는 결제수단을 비활성화하되, 활성 구독이 1개 이상 있는 회원은 공통 결제수단이 최소 1개 남아야 합니다.
