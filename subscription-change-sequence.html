구독 플랜 변경 플로우

회원이 현재 상품 구독 안에서 플랜을 변경할 때 즉시 업그레이드는 차액을 청구하고, 다운그레이드나 다음 주기 변경은 예약 상태로 저장한 뒤 다음 정기 결제부터 반영하는 흐름입니다.

전체 시퀀스 목록전체 API 목록API 상세 설명

사용 API

POST /subscriptions/{subscriptionId}/change-preview

목표 플랜 변경이 업그레이드인지 다운그레이드인지 서버가 판정하고, 유저 확인에 필요한 결제 금액과 적용일을 계산합니다.

PATCH /subscriptions/{subscriptionId}

플랜 변경 미리보기에서 발급한 확인 토큰으로 업그레이드는 즉시 변경하고 다운그레이드는 다음 결제일 반영으로 예약합니다.

GET /subscriptions/me

현재 회원의 상품별 구독 목록, 이용 기간, 다음 결제일, 공통 기본 결제수단 표시 정보를 조회합니다.

POST /internal/subscription-billing/run

다음 결제일이 도래한 구독을 찾아 정기 자동결제를 실행합니다.

POST /webhooks/toss-payments

토스페이먼츠 결제 이벤트를 수신하고 내부 결제/구독 상태를 보정합니다.

즉시 업그레이드 및 차액 결제

상위 플랜으로 즉시 변경하는 경우 남은 기간의 차액을 계산하고 저장된 빌링키로 추가 결제를 승인한 뒤 구독 권한을 바로 갱신합니다.

다음 주기 플랜 변경 예약

다운그레이드 또는 결제 주기 변경처럼 즉시 결제하지 않는 변경은 현재 기간 종료일까지 기존 권한을 유지하고 다음 정기 결제부터 새 플랜을 적용합니다.
