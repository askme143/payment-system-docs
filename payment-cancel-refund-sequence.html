결제 취소/환불 플로우

승인 완료된 일반결제를 회원 또는 운영자가 전체/부분 취소하고, 토스 취소 API 결과를 내부 결제 상태와 취소 이력에 반영하는 흐름입니다.

전체 시퀀스 목록전체 API 목록API 상세 설명MongoDB 구조 문서

사용 API

POST /payments/{paymentId}/cancel

승인 완료된 일반결제를 전체 또는 부분 취소하고 내부 취소 이력을 저장합니다.

GET /admin/payments

CS와 운영자가 결제 목록을 검색합니다.

POST /admin/payments/{paymentId}/cancel

관리자가 회원 요청, 중복 결제, 장애 보정 사유로 일반결제를 전체 또는 부분 취소합니다.

전체 취소 및 부분 취소

회원이 마이페이지에서 일반결제 취소를 요청했을 때 취소 가능 금액을 검증하고 토스 취소 API를 호출한 뒤 내부 상태를 canceled 또는 partial_canceled로 저장합니다.

상태 요약

시점구독 상태결제 상태설명
취소 전-paid승인 완료 결제이며 cancelableAmount가 남아 있습니다.
부분 취소 후-partial_canceled일부 금액만 취소됐고 남은 금액은 추가 취소할 수 있습니다.
전체 취소 후-canceled취소 가능 금액이 0이며 추가 취소 요청은 멱등 결과 또는 409로 처리합니다.
운영자 취소 전-paid 또는 partial_canceled관리자가 취소 가능 금액과 사유를 확인합니다.
운영자 취소 후-canceled 또는 partial_canceled회원 결제 내역과 운영 감사 로그가 같은 취소 건을 가리킵니다.
