정기 결제 실행 플로우

스케줄러가 다음 결제일이 도래한 활성 구독을 상품 구독 단위로 잠그고 인보이스를 생성한 뒤 공통 기본 결제수단으로 토스 자동결제를 승인하고 다음 결제일을 예약하는 운영 흐름입니다.

전체 시퀀스 목록전체 API 목록API 상세 설명MongoDB 구조 문서

사용 API

POST /internal/subscription-billing/run

다음 결제일이 도래한 구독을 찾아 정기 자동결제를 실행합니다.

POST /webhooks/toss-payments

상태 요약

시점구독 상태결제 상태설명
사전 알림 대상 선정activescheduled다음 결제일 7일 전인 활성 구독을 대상으로 알림 데이터를 구성합니다.
이메일 발송 후activescheduled결제일과 결제 금액이 포함된 사전 알림 발송 이력을 저장합니다.
배치 대상 선정 후activepending다음 결제일이 도래한 구독이 잠기고 청구 회차가 생성됩니다.
자동결제 성공 후activepaid인보이스와 결제 레코드가 성공 처리되고 다음 결제일이 예약됩니다.
웹훅 보정 후activepaid토스 이벤트와 내부 상태가 일치하도록 멱등 보정됩니다.
