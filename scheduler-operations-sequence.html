스케쥴러 운영 플로우

Kubernetes CronJob이 같은 payments 이미지를 1회 실행하고, 어드민 콘솔이 스케쥴러 실행 이력 조회와 수동 실행을 제공하는 흐름입니다.

전체 시퀀스 목록전체 API 목록API 상세 설명이메일 발송 시스템 아키텍처 문서MongoDB 구조 문서결제 시스템 잠재 위험 및 대응 문서

사용 API

GET /admin/scheduler-runs

운영자가 스케쥴러 실행 이력을 검색합니다.

GET /admin/scheduler-runs/{runId}

운영자가 특정 스케쥴러 실행의 상세 결과와 오류 요약을 조회합니다.

POST /admin/scheduler-runs

운영자가 어드민 콘솔에서 스케쥴러 작업을 수동 실행합니다.

연관 잠재 위험
  • recurring-billing-duplicate-run 정기결제 배치 중복 실행
  • idempotency-store-missing 멱등키 저장소 부재
  • cancel-scheduled-billing-race 해지 예약과 정기 과금 경합
