사용자 users

이미 존재하는 사용자 계정 컬렉션입니다. 결제 도메인은 user_id로만 참조하고 결제 관련 상태는 저장하지 않습니다.

필드

필드타입필수설명상세
_idObjectId예사용자 문서 ID입니다.-
emailstring예로그인과 결제 안내에 사용하는 이메일입니다.-
statusstring예사용자 계정 상태입니다.enum: active, withdrawn, blocked

인덱스

필드유니크설명
email예이메일 중복 가입을 방지합니다.

관련 API

  • POST /subscriptions/checkout - 구독 빌링 인증 전에 구독 대기 레코드와 토스 SDK 호출 데이터를 생성합니다.
  • POST /payments/orders - 일반결제 결제창 호출 전에 내부 주문을 생성합니다.
  • POST /billing/auth - 기존 회원이 결제수단을 변경할 때 빌링 인증 시작 데이터를 생성합니다.
  • POST /billing/issue - 빌링 인증 성공 후 새 빌링키를 발급받아 결제수단으로 저장합니다.

상품 products

구독 플랜과 일회성 SKU가 속한 상품군과 판매 상태를 관리합니다.

필드

필드타입필수설명상세
_idObjectId예상품 문서 ID입니다.-
product_codestring예외부 노출과 API 필터에 사용하는 상품 코드입니다.-
product_typestring예구독 상품인지 일반결제 상품인지 구분합니다.enum: subscription, one_time
namestring예관리자와 사용자 화면에 표시할 상품명입니다.-
statusstring예상품 판매 및 노출 상태입니다.enum: draft, active, paused, archived

인덱스

필드유니크설명
product_code, product_type예상품 코드와 유형 조합을 고유하게 유지합니다.

관련 API

  • GET /plans - 구매 가능한 상품과 구독 플랜 목록을 조회합니다.
  • GET /plans/{planId} - 단일 플랜의 가격, 주기, 판매 가능 상태를 조회합니다.
  • POST /admin/products - 관리자가 공통 Product를 생성하고 구독상품 또는 일반상품 타입을 지정합니다.
  • PATCH /admin/products/{productId}/status - 상품을 초안, 판매중, 판매중지, 아카이브 상태로 전환합니다.

구독 플랜 subscription_plans

구독 상품의 과금 주기, 금액, 권한, 판매 가능 상태를 관리합니다.

필드

plan_codestring예상품 안에서 플랜을 구분하는 코드입니다.-

amountnumber예플랜 정가입니다.-

관련 API

  • GET /plans - 구매 가능한 상품과 구독 플랜 목록을 조회합니다.
  • GET /plans/{planId} - 단일 플랜의 가격, 주기, 판매 가능 상태를 조회합니다.
  • POST /admin/products/{productId}/subscription-plans - 구독상품에 월간, 연간 같은 구독 플랜을 추가합니다.
  • PATCH /admin/products/{productId}/subscription-plans/{planId} - 구독 플랜의 표시 정보, 판매 상태, 신규 가입 가격 정책을 수정합니다.
  • POST /subscriptions/{subscriptionId}/change-preview - 목표 플랜 변경이 업그레이드인지 다운그레이드인지 서버가 판정하고, 유저 확인에 필요한 결제 금액과 적용일을 계산합니다.
  • PATCH /subscriptions/{subscriptionId} - 플랜 변경 미리보기에서 발급한 확인 토큰으로 업그레이드는 즉시 변경하고 다운그레이드는 다음 결제일 반영으로 예약합니다.

구독 상태

subscriptions.status

pending, active, past_due, cancel_scheduled, canceled

전이이벤트
pending → active첫 결제 승인과 빌링키 저장이 완료됩니다.
active → cancel_scheduled사용자가 현재 이용기간 종료 해지를 예약합니다.
active → past_due정기결제 실패 후 재시도 대상이 됩니다.
cancel_scheduled → active사용자가 해지 예약을 철회합니다.
cancel_scheduled → canceled이용기간 종료 시각에 도달합니다.

payments.status

ready, paid, failed, canceled, partial_canceled

전이이벤트
ready → paid토스 결제 승인에 성공합니다.
ready → failed인증 실패, 승인 실패, 정기결제 실패가 기록됩니다.
paid → canceled전체 취소가 완료됩니다.
paid → partial_canceled부분 취소가 완료됩니다.
