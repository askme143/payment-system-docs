작성 완료

구독

구독 시작, 변경, 해지와 재개처럼 회원 구독 생애주기를 다루는 플로우입니다.

available

신규 회원 최초 구독 성공 플로우

체크아웃 생성, 빌링 인증, 빌링키 발급, 첫 자동결제 승인, 웹훅 보정까지의 성공 흐름입니다.

  • GET /plans
  • GET /plans/{planId}
  • POST /subscriptions/checkout
  • POST /subscriptions/confirm
  • POST /webhooks/toss-payments
  • GET /subscriptions/me
available

신규 회원 최초 구독 실패 플로우

체크아웃 실패, 토스 빌링 인증 실패, 구독 확정 검증 실패, 빌링키 발급 실패, 첫 결제 실패, 웹훅 실패를 나눠 설명합니다.

  • GET /plans
  • GET /plans/{planId}
  • POST /subscriptions/checkout
  • POST /subscriptions/confirm
  • POST /webhooks/toss-payments
available

구독 해지 플로우

회원이 기간 종료 시 해지 예약을 요청하면 현재 이용 기간은 유지하고, 기간 종료 이후 만료 배치가 구독을 canceled로 최종 마감하는 흐름입니다.

  • POST /subscriptions/{subscriptionId}/cancel
  • POST /subscriptions/{subscriptionId}/resume
  • GET /subscriptions/me
  • POST /internal/subscription-billing/run
available

구독 재개 플로우

기간 종료 전 해지 예약을 철회해 구독을 active 상태로 되돌리고 다음 정기 결제일을 복구하는 흐름입니다.

  • POST /subscriptions/{subscriptionId}/resume
  • GET /subscriptions/me
  • POST /internal/subscription-billing/run
available

구독 플랜 변경 플로우

회원이 현재 상품 구독 안에서 플랜을 변경할 때 즉시 업그레이드는 차액을 청구하고, 다운그레이드나 다음 주기 변경은 예약 상태로 저장한 뒤 다음 정기 결제부터 반영하는 흐름입니다.

  • GET /plans
  • GET /plans/{planId}
  • POST /subscriptions/{subscriptionId}/change-preview
  • PATCH /subscriptions/{subscriptionId}
  • GET /subscriptions/me
  • POST /internal/subscription-billing/run
  • POST /webhooks/toss-payments

정기 결제/재시도

정기 과금 실행, 실패 재시도, 최종 실패 종료처럼 배치와 청구 상태 전이를 다루는 플로우입니다.

available

정기 결제 실행 플로우

스케줄러가 다음 결제일이 도래한 활성 구독을 상품 구독 단위로 잠그고 인보이스를 생성한 뒤 공통 기본 결제수단으로 토스 자동결제를 승인하고 다음 결제일을 예약하는 운영 흐름입니다.

  • POST /internal/subscription-billing/run
  • POST /webhooks/toss-payments
available

정기 결제 실패 및 재시도 플로우

정기 결제 승인 실패 후 실패 사유를 저장하고 재시도 일정을 예약한 뒤, 재시도 성공 또는 최종 실패에 따라 구독 상태를 보정하는 흐름입니다.

  • POST /internal/subscription-billing/{invoiceId}/retry
  • GET /invoices/{invoiceId}
  • POST /webhooks/toss-payments
available

구독 결제 최종 실패 후 구독 종료 플로우

정기 결제 재시도가 모두 실패하면 별도 유예 기간 없이 구독을 canceled로 종료하고, 다시 이용하려면 새 구독을 시작하도록 안내하는 흐름입니다.

  • GET /invoices/{invoiceId}
  • POST /internal/subscription-billing/{invoiceId}/retry

청구/결제수단

청구 내역 조회와 결제수단 등록, 기본값, 삭제 정책을 다루는 플로우입니다.

available

마이페이지 청구 내역 조회 플로우

회원이 마이페이지 청구 내역에서 인보이스 목록을 조회하고, 상세 화면에서 결제 성공 영수증 또는 결제 실패 사유와 재시도 안내를 확인하는 흐름입니다.

  • GET /invoices
  • GET /invoices/{invoiceId}
available

결제수단 관리 플로우

회원이 결제 수단을 추가하고, 모든 상품 구독이 함께 사용하는 공통 기본 결제수단을 지정하며, 활성 구독이 1개 이상 있으면 최소 1개의 결제수단을 유지하도록 삭제를 제한하는 흐름입니다.

  • POST /billing/auth
  • POST /billing/issue
  • GET /billing/methods
  • PATCH /billing/methods/{billingMethodId}/default
  • DELETE /billing/methods/{billingMethodId}

일반결제

단건 결제 생성, 실패 재시도, 결제 취소와 환불을 다루는 플로우입니다.

available

일반결제 플로우

구매 의도 생성, 결제 시도 생성, 토스 결제창 인증, 결제 승인, 웹훅 보정까지의 일반결제 성공 흐름입니다. 실패 후 재시도는 기존 시도를 재사용하지 않고 새 paymentId와 orderId를 발급합니다.

  • POST /payments/orders
  • POST /payments/confirm
  • GET /payments/{paymentId}
  • POST /webhooks/toss-payments
available

일반결제 실패 및 재시도 플로우

주문 생성 실패, 사용자 취소, 결제수단 인증 실패, 승인 전 서버 검증 실패, 토스 승인 실패, 실패 후 다른 결제수단 재시도를 나눠 설명합니다.

  • POST /payments/orders
  • POST /payments/confirm
  • GET /payments/{paymentId}
  • POST /payments/{paymentId}/auth-result
  • POST /webhooks/toss-payments
available

결제 취소/환불 플로우

승인 완료된 일반결제를 회원 또는 운영자가 전체/부분 취소하고, 토스 취소 API 결과를 내부 결제 상태와 취소 이력에 반영하는 흐름입니다.

  • POST /payments/{paymentId}/cancel
  • GET /admin/payments
  • POST /admin/payments/{paymentId}/cancel

어드민/운영

관리자 상품 관리와 운영자 수동 보정처럼 내부 운영 화면에서 다루는 플로우입니다.

available

어드민 상품 관리 플로우

관리자가 공통 Product를 만들고, 구독상품은 구독 플랜으로, 일반상품은 단건 구매 SKU로 분리해 판매 조건과 상태를 관리하는 흐름입니다.

  • POST /admin/products
  • POST /admin/products/{productId}/subscription-plans
  • PATCH /admin/products/{productId}/subscription-plans/{planId}
  • POST /admin/products/{productId}/one-time-skus
  • PATCH /admin/products/{productId}/one-time-skus/{skuId}
  • PATCH /admin/products/{productId}/status
available

운영자 구독 수동 보정

관리자가 토스 결제 성공 누락, 웹훅 처리 실패, CS 예외, 재시도 실패 후 별도 결제, 마이그레이션 오류를 감사 가능한 API로 보정하는 흐름입니다.

  • GET /admin/subscriptions
  • POST /admin/subscriptions/{subscriptionId}/adjust
