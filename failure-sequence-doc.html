Sequence Diagram

신규 회원 최초 구독 실패 플로우

체크아웃 실패, 토스 빌링 인증 실패, 구독 확정 검증 실패, 빌링키 발급 실패, 첫 결제 실패, 웹훅 실패를 나눠 설명합니다.

전체 시퀀스 목록전체 API 목록API 상세 설명MongoDB 구조 문서

사용 API

GET /plans

구매 가능한 상품과 구독 플랜 목록을 조회합니다.

GET /plans/{planId}

단일 플랜의 가격, 주기, 판매 가능 상태를 조회합니다.

POST /subscriptions/checkout

구독 빌링 인증 전에 구독 대기 레코드와 토스 SDK 호출 데이터를 생성합니다.

POST /subscriptions/confirm

프론트 성공 페이지가 호출하며, 빌링키 발급과 첫 결제로 구독을 활성화합니다.

POST /webhooks/toss-payments

토스페이먼츠 결제 이벤트를 수신하고 내부 결제/구독 상태를 보정합니다.
