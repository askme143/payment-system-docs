Sequence Diagram

어드민 상품 관리 플로우

관리자가 공통 Product를 만들고, 구독상품은 구독 플랜으로, 일반상품은 단건 구매 SKU로 분리해 판매 조건과 상태를 관리하는 흐름입니다.

전체 시퀀스 목록전체 API 목록API 상세 설명MongoDB 구조 문서결제 시스템 잠재 위험 및 대응 문서

사용 API

POST /admin/products

관리자가 공통 Product를 생성하고 구독상품 또는 일반상품 타입을 지정합니다.

POST /admin/products/{productId}/subscription-plans

구독상품에 월간, 연간 같은 구독 플랜을 추가합니다.

PATCH /admin/products/{productId}/subscription-plans/{planId}

구독 플랜의 표시 정보, 판매 상태, 신규 가입 가격 정책을 수정합니다.

POST /admin/products/{productId}/one-time-skus

일반상품에 단건 구매용 SKU와 판매 가격, 재고 정책을 추가합니다.

PATCH /admin/products/{productId}/one-time-skus/{skuId}

일반상품 SKU의 가격, 재고 정책, 판매 상태를 수정합니다.

PATCH /admin/products/{productId}/status

상품을 초안, 판매중, 판매중지, 아카이브 상태로 전환합니다.

상태 요약

시점구독 상태결제 상태설명
초안 구독상품 생성draft product-공통 Product가 생성되지만 플랜이 없으면 공개 /plans에는 노출하지 않습니다.
구독 플랜 활성화active planrecurring ready신규 구독과 플랜 변경 플로우가 같은 productCode 안의 활성 플랜만 선택할 수 있습니다.
초안 일반상품 생성-draft product공통 Product가 생성되지만 SKU가 없으면 일반결제 주문 생성에서 선택할 수 없습니다.
SKU 활성화-one_time purchasable일반결제 주문 생성 API는 활성 SKU만 조회하고 서버 가격으로 amount를 다시 계산합니다.
