정기 결제 실패 및 재시도 플로우

정기 결제 승인 실패 후 실패 사유를 저장하고 재시도 일정을 예약한 뒤, 재시도 성공 또는 최종 실패에 따라 구독 상태를 보정하는 흐름입니다.

전체 시퀀스 목록전체 API 목록API 상세 설명MongoDB 구조 문서결제 시스템 잠재 위험 및 대응 문서

사용 API

POST /internal/subscription-billing/{invoiceId}/retry

실패한 구독 결제를 수동 또는 배치로 재시도합니다.

GET /invoices/{invoiceId}

단일 인보이스의 금액, 결제 시도, 실패 사유, 영수증 정보를 조회합니다.

POST /webhooks/toss-payments

토스페이먼츠 결제 이벤트를 수신하고 내부 결제/구독 상태를 보정합니다.

연관 잠재 위험
  • subscription-retry-duplicate-charge 실패 인보이스 재시도 중복 과금
  • idempotency-store-missing 멱등키 저장소 부재

상태 요약

시점구독 상태결제 상태설명
자동결제 실패 직후past_duefailed실패 인보이스와 결제 시도가 저장되고 재시도 가능 여부를 판단합니다.
재시도 예약 후past_dueretry_scheduled다음 재시도 시각이 예약되고 결제 실패 상태가 유지됩니다.
실패 웹훅 보정 후past_duefailed토스 이벤트가 늦게 도착해도 내부 실패 상태를 멱등하게 유지합니다.
재시도 시작 전past_duefailed실패 회차가 재시도 가능 상태이며 중복 실행을 막기 위해 잠깁니다.
재시도 성공 후activepaid결제 실패 상태가 해제되고 다음 정기 결제일이 정상 갱신됩니다.
최종 실패 후suspendedfinal_failed서비스 이용을 제한하고 회원에게 결제수단 변경 또는 재개를 안내합니다.
